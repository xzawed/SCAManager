### 분할 모델별 평가

| 모델 | 적용 시기 | ROI | 권장 사용처 |

| **cleanup 4 갈래 (코드/문서/polish/가드)** | 결함이 4 책임 영역 걸침 | ★★★★ | 정합성 감사 후속 |

| **단일 PR 일괄** | 단일 영역 + 결함 ≤ 5 | ★★★ | 작은 fix |

### 회피해야 할 안티 패턴

1. **동일 docs 파일 PR 병렬 진행** (PR-5 머지 전 PR-D1~D3) — 본 사이클은 운 좋게 conflict 0
2. **PR push 전 `git log -5` 미확인** → 중복 머지 3건. 1줄 명령으로 100% 예방 가능
3. **단일 작업일 25+ PR + 사람 검토 의존** — 검토 누락 ∝ PR 수
4. **5-way sync 단언이 회귀 가드 없이 머지** → 사고 시 책임 귀속 어려움

**새 default 패턴** (사용자 합의):

**이유** (사용자 발화): *"반대 하기를 원하기 보다 각각의 장단점을 알려주시면 권장을 한다 하더라도 제가 판단하는데 도움이 될거라 생각합니다."*

**Claude 의 추가 약속**:

- 옵션 표 마지막에 "고려했으나 제시 안 한 안" 1줄 명시 (추론 추적성)

### 정책 2: PR 본문 **"🔍 사용자 검증 필요" 섹션 의무**

이전: PR 본문이 "tests pass" 로 끝남
다음: 사용자가 시각/운영 확인할 항목 1~3개 명시

```markdown
## 🔍 사용자 검증 필요
- [ ] Railway 배포 후 `/repos/{owner}/{repo}/settings` 데스크탑 + 모바일 확인
- [ ] claude-dark 테마 토글 시 카드 헤더 정상 표시
- [ ] (있다면) 운영 사고 보고
```

### 정책 3: **자율 판단 결정의 사후 보고 의무**

위임받은 작업 ("권장 방향 진행" 등) 중 Claude 가 판단한 항목은 PR 본문 또는 응답 끝에 명시.

예: "GitHub 정리 시 미매핑 PR 17건은 보류 결정했습니다 (역사 정리 가치 < 시간 부담). 이의 있으시면 알려주세요."

### 정책 4: **단언과 회귀 가드를 같은 PR 에 묶기**

"5-way sync 보존" 같은 단언은 회귀 가드 PR 과 동시 머지. 사고 시 책임 귀속 명확화.

### 정책 5: **사이클 종료 신호 명시**

"권장 진행" 시작된 작업의 종료 시점이 모호. 사용자 신호:
- "여기까지 합시다" / "오늘은 그만" — 명시 종료
- 신호 없으면 Claude 가 사이클 끝마다 "다음 단계 진행할까요?" 명시 확인

### 정책 6 (보조): **다중 에이전트 디스패치 시 `line:span` 인용 의무**

본 사이클 검증 — 인용 없는 보고는 false-positive 80% 발생. 모든 에이전트 프롬프트 강제 조건.

### 정책 7: **모든 작업은 PR 단위 (main 직접 작업 지양)**

사용자 발화: *"작업 단위를 PR 로 수행해주시길 권장합니다. 가급적 main 에 직접 작업은 지양해 주셨으면 합니다."*

**default 작업 흐름**:
1. `git checkout main && git pull origin main`
2. `git checkout -b <type>/<scope>-<short-desc>`
3. 작업 + commit
4. `git push -u origin <branch>` → PR 생성

**예외 0** — docs only / typo / 사소한 정정 모두 PR. 예외 만들면 정책 일관성 깨짐.

**위반 시 회복** (main 에 실수 commit 시):
```bash
git branch <type>/<scope>-<desc>     # 새 브랜치로 commit 이동
git reset --hard origin/main         # main reset
git checkout <type>/<scope>-<desc>
git push -u origin <branch>
```

(2026-05-01 본 사이클은 모든 작업이 PR 로 진행됐으나, 본 정책 명시로 향후 세션 이탈 차단.)

### 정책 8: **회고는 항상 다중 에이전트로 깊게 진행 의무**

사용자 발화: *"가급적 회고도 모든 에이전트가 깊게 생각하여 진행했으면 합니다."*

**default 회고 패턴**:
- 최소 4~5 에이전트 병렬 디스패치
- 관점 분리 (작업 패턴 / 다중 에이전트 운영 / 사용자 협업 / 기술 학습 / 문서 정합성)
- 각 에이전트 프롬프트: self-contained + line:span 인용 의무 + 출력 포맷 강제

**검증된 효과** (본 사이클): 단일 Claude 회고 vs 5-에이전트 회고 = +27 결함 식별. 단일 감사는 누락 위험 — 2단계가 표준.

**금지**: Claude 단독 회고 / 1~2 에이전트만 디스패치.
