# slither (Solidity 정적분석) Implementation Plan — Phase D.2

> **Status:** 📝 **계획 수립 완료** (2026-04-21). 스펙 [docs/superpowers/specs/2026-04-21-slither-design.md](../specs/2026-04-21-slither-design.md) 승인 후 착수.
>
> **실행 환경**: pytest 호출 시 `.env` stash + env unset 래퍼 사용 (모든 pytest 블록 동일).

**Goal:** Solidity 전용 정적분석 도구 `slither` 를 Analyzer Protocol 로 추가. Semgrep 과 독립적으로 smart-contract 특유 취약점(reentrancy/suicidal/arbitrary-send 등) 을 탐지. 신규 파일 1개(`src/analyzer/tools/slither.py`) + `static.py` import 1줄 + `requirements.txt` 1줄 + 테스트 파일 1개.

**Architecture:** cppcheck/shellcheck 패턴 그대로 재사용. `_SlitherAnalyzer` 가 `Analyzer` Protocol 을 구현, `register()` 로 전역 REGISTRY 자동 등록. slither 바이너리가 없으면 `is_enabled()=False` → 조용히 skip. Mixed-category(security/code_quality) 지원 — detector name 화이트리스트 기반 분류.

**Tech Stack:** Python 3.12, subprocess, stdlib json, slither-analyzer 0.10+ (pip), solc 런타임 (slither 가 자동 관리).

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract Vault {
    mapping(address => uint) public balances;

    function withdraw(uint amount) public {
        require(balances[msg.sender] >= amount);
        (bool ok, ) = msg.sender.call{value: amount}("");
        require(ok);
        balances[msg.sender] -= amount;  // 🚨 상태 변경이 external call 뒤에 옴

}
